# DeepForest Microgreen Plant Detection Guide

Complete guide for training, tuning, and deploying the DeepForest-based plant counting system.

---

## 1. Environment Setup

### Prerequisites
- Python 3.9+
- pip

### Installation

```bash
# Navigate to project root
cd "/path/to/Microgreen-App"

# Install DeepForest and dependencies
pip install deepforest pytorch-lightning torch torchvision
pip install pandas numpy opencv-python-headless

# If you have a GPU (CUDA), install PyTorch with CUDA support:
# pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

### Required Packages Summary
| Package | Purpose |
|---------|---------|
| `deepforest` | Pretrained tree crown detection model (RetinaNet) |
| `pytorch-lightning` | Training framework used by DeepForest |
| `torch` / `torchvision` | Deep learning backend |
| `pandas` | Data handling (CSV annotations) |
| `opencv-python-headless` | Image I/O and visualization |
| `numpy` | Array operations |

## 3. Quick Start

### Step 1: Convert Data
```bash
cd ml_engine
python convert_coco_to_deepforest.py
```

### Step 2: Train Model
```bash
python train_deepforest.py
# Or with custom settings:
python train_deepforest.py --epochs 100 --lr 0.00005 --batch-size 2
```

### Step 3: Run Inference
```bash
python inference_deepforest.py --image path/to/tray.jpg
```

### Step 4: Start Backend
```bash
cd ../backend
python -m uvicorn app.main:app --reload --port 8000
# Upload image to POST /api/count-plants
```

## 4. How to Retrain When New Data Is Added

1. **Add new annotated images** to your Roboflow project and re-export in COCO format
2. **Update the dataset path** in `config_deepforest.py`:
   ```python
   COCO_DATASET_DIR = Path(r"path/to/new/export/train")
   ```
3. **Re-run the converter** to regenerate CSVs:
   ```bash
   python convert_coco_to_deepforest.py
   ```
4. **Retrain** (optionally from the previous checkpoint):
   ```bash
   python train_deepforest.py --epochs 30
   ```
5. **Restart the backend** to load the new model

> **Tip**: More training data = better accuracy. Aim for 100+ annotated images for robust performance.

## 5. Hyperparameter Tuning for Dense Plant Trays

### Most Impactful Parameters

| Parameter | Default | Tuning Strategy |
|-----------|---------|-----------------|
| `PATCH_SIZE` | 400 | **Decrease to 200-300** for very dense/small plants. Smaller patches = better small-object detection |
| `PATCH_OVERLAP` | 0.3 | **Increase to 0.4-0.5** if plants at patch borders are missed |
| `SCORE_THRESHOLD` | 0.3 | **Lower to 0.15-0.2** to catch more faint plants; raise to 0.4-0.5 to reduce false positives |
| `IOU_THRESHOLD` | 0.3 | **Lower to 0.2** if duplicate detections appear; raise to 0.4 if overlapping plants merge |
| `LEARNING_RATE` | 0.0001 | **Lower to 0.00005** for small datasets (<50 images) to avoid overfitting |
| `EPOCHS` | 50 | **Increase to 100-200** for better convergence; monitor validation loss |
| `BATCH_SIZE` | 4 | **Decrease to 1-2** if GPU memory is limited |

### Recommended Tuning Process

1. **Start with defaults** and evaluate on a few test images
2. **If under-counting**: Lower `SCORE_THRESHOLD`, decrease `PATCH_SIZE`, increase `PATCH_OVERLAP`
3. **If over-counting** (duplicates): Lower `IOU_THRESHOLD`, increase `SCORE_THRESHOLD`
4. **If very small plants are missed**: Use `PATCH_SIZE=200` with `PATCH_OVERLAP=0.5`
5. **If training loss plateaus early**: Lower `LEARNING_RATE`, increase `EPOCHS`

## 6. Improvement Suggestions

### If Bounding Box Detection Struggles

1. **Instance Segmentation with SAM (Segment Anything Model)**
   - Use SAM to generate pixel-level masks instead of boxes
   - Better for overlapping/touching plants
   - Integration: Use DeepForest for initial detection, then SAM for refinement

2. **CenterNet / Point-Based Detection**
   - Instead of bounding boxes, predict plant center points
   - More natural for circular plant crowns
   - Libraries: `mmdetection` with CenterNet config

3. **Density Estimation**
   - Skip individual detection entirely
   - Train a regression model to predict plant count from image patches
   - Works well for extremely dense trays where plants overlap heavily
   - Libraries: `CrowdCounting`, CSRNet

4. **Data Augmentation Pipeline**
   - Add `albumentations` for aggressive augmentation:
     - Random crop, flip, rotate
     - Color jitter, brightness/contrast
     - Mosaic augmentation (combine 4 images)
   - Especially important with small datasets

5. **Active Learning Loop**
   - Use the current model to pre-annotate new images
   - Manually correct mistakes
   - Retrain with expanded dataset
   - Repeat until accuracy plateaus
